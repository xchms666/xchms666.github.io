加法指令：ADD, INC, ADC

INC

INC与ADD比较

  • INC速度比较快
  • ADD影响CF（进位）标志位

inc操作后CF仍然保持原值

减法指令：SUB SBB DEC NEG CMP

DEC

decrement 自减，dec指令不影响CF

SBB

subtract with borrow 带借位减

应用举例

大数的减法运算

NEG

negate 求相反数

会影响CF,ZF,SF等标志位

CMP

cmp与sub的区别是cmp抛弃两数之差，仅保留标志位状态

比较跳转指令

针对非符号数

ja jb jae jbe

依据CF和ZF的状态

针对符号数

jg jl jge jle

依据SF和OF的状态

特殊情况

  1. 手动改sf或of或zf
  2. 两个负数相加
    1. 但在减法跳转指令中不存在

SF表示结果为正

OF表示有无溢出

小数运算

fadd fsub fmul fdiv

运算后是IEE754格式，要转化后才能输出

由浮点处理器负责执行, 用法请参考主页intel指令集

小数变量的定义:

pi dd 3.14; 32位小数,相当于float
r  dq  3.14159; 64位小数, 相当于double
; q:quadruple 4倍的
s  dt  3.14159265; 80位小数, 相当于long double

在C语言中要输出long double的值需要使用"%Lf"格式

CPU内部一共有8个小数寄存器，分别叫做

st(0)、st(1)、…、st(7)

其中st(0)简称st

这8个寄存器的宽度均达到80位，相当于C语言中的

long double类型。

VC里面的long double类型已经退化成double类型。

举例

;Turbo Debugger跟踪时，
;点菜单View->Numeric Processor查看小数堆栈
data segment
abc dd 3.14
xyz dd 2.0; 若载入整数，fld->fild
result dd 0
data ends
code segment
assume cs:code, ds:data
main:
   mov ax, data
   mov ds, ax
   fld abc; 把3.14载入到小数堆栈-->st(0)-->st(1)fld会自动识别后跟参数的宽度，编译器会自动加上
   fld xyz; 把2.0载入到小数堆栈-->st(0)
   fmul st, st(1); 两数相乘st(0)=st(0)*st(1)
   fstp result; 保存结果到result，并弹出
   fstp st      ; 弹出小数堆栈中残余的值
   mov ah, 4Ch
   int 21h
code ends
end main
