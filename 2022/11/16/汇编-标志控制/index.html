汇编_控制位

Created2022-11-16|Updated2022-11-16|汇编
|Post View:

标志位

状态标志

FL标志寄存器

结构

CF：进位标志

作用

应用示例

  • 加减有关

mov ah, 0FFh

add ah, 1; AH=0, CF=1产生了进位

add ah, 2; AH=2, CF=0

sub ah, 3; AH=0FFh, CF=1产生了借位

  • 移位

mov ah, 10110110B

shr ah, 2；该语法要求源代码最前面加.386

​ ；并且每个段定义时segement后跟use16

​ ；CF=1

与CF相关的两条跳转指令:

jc

​ 有进位则跳转

jnc

​ 无进位则跳转

adc

​ 带进位加，可用于二进制的转换

clc clean carry flag

​ CF=0

stc set carry flag

​ CF=1

ZF 零标志(zero flag)

sub ax, ax; AX=0, ZF=1

add ax, 1; AX=1, ZF=0

add ax, 0FFFFh; AX=0, ZF=1, CF=1

jz is_zero; 会发生跳转, 因为当前ZF==1

  • 与jz相反的指令是jnz, jnz是根据ZF==0作出跳转

  • jz=je，jnz=jne

    • CPU在做cmp指令时，会做减法运算
    • 若相减等于0，ZF==1，je跳转，相当于jz
    • 同理jnz=jne

  • xor ax，ax

    jz/je equal

    • 会跳转，因为ZF==1
    • 但je看起来不合逻辑

注意：mov指令不影响任何标志位

mov ax, 1234h               
mov bx, 1234h               
sub ax, bx                      
mov bx, 1;此mov不影      
           ;响sub指令
           ;产生的ZF状态
jz iszero
mov bx, 0               
iszero:
  • mov非运算指令，不改变ZF的值
mov ax, 1234h
mov bx, 1234h               
sub ax, bx                      
jz iszero
mov bx, 0
jmp done; 与左边相比
; 这里多出一条
; jmp指令
iszero:    ; 故左边写法更好
mov bx, 1
done:

PF

  • 只统计第八位中1的个数
    • 偶数个1——1
    • 奇数个1——0

相关指令

  • jp
    • 当PF==1时则跳
    • jp也可以写成jpe(jump if parity even)
  • jnp
    • 当PF==0时则跳
    • jnp也可以写成jpo(jump if parity odd)
  • 用于数据传输的校验（串口
    • 奇校验: 数据位+校验位合起来，1的个数必须是奇数
    • 偶校验: 数据位+校验位合起来，1的个数必须是偶数
    • 举例

AF (Auxiliary Flag) 辅助进位标志

低4位向高4位产生进位或借位 或 第三位向第四位进位或借位时 AF=1

  • AF跟BCD(Binary Coded Decimal)码有关
    • 一般不用除法来获取十位数的十位和个位因为除法不好用，慢
    • 举例
      • 29h
        • 29h >> 4 = 02h
        • 29h & 0Fh = 09h
    • daa指令
      • 29h+08 产生AF=1情况
      • 29h+01 不产生AF=1情况
mov al, 29h; 分钟
add al, 08 ; 过了8分钟
             ; 31h
daa; decimal adjust for addition加法的十进制调整
; 这条指令会根据AF=1或(AL & 0Fh)>9，做以
; 下运算: AL = AL + 6
; AL=37h

mov al, 29h
add al, 1; AL=2Ah
daa; AL=AL+6=2Ah+6=30h

跳转指令

无！

总结

CF ZF SF OF AF PF: 这6个称为状态标志

DF TF IF: 这3个称为控制标志

DF: direction flag

TF: trace/trap flag

IF: interrupt flag

控制标志

DF(Direction Flag)方向标志: 控制字符串的操作方向

  • 当DF=0时为正方向(低地址到高地址)，当DF=1是反方向
    • 顺着复制 ERROR
    • 倒着复制
  • 复制规范
    • 若源首地址<目标首地址，则复制按反方向
    • 当源首地址>目标首地址时，复制时按正方向

相关指令

  • cld指令使DF=0， std指令使DF=1
  • rep movsb；repeat move string by byte

相关C语言函数

strcpy(target, source); 永远按正方向复制

  • 若source和target有部分重叠就可能出问题！

memcpy(target, source, n);永远按正方向复制

memmove(target, source, n); 能正确处理部分重叠

IF（Interrupt Flag）中断标志

当IF=1时,允许中断;否则禁止中断

相关指令

  • cli指令使IF=0表示关/禁止硬件中断
  • sti指令使IF=1表示开/允许硬件中断
    • 硬件中断常见
      • 键盘中断int 09h
      • 时钟中断int 08h
        • 定时中断请求

举例

cli
;十万次循环，中途CPU不处理键盘中断
sti
;CPU会接受键盘中断
  • 相当于一组括号，使内部内容进入免打扰状态
  • 多线程：同一时刻运行多个函数

TF（Trace/Trap Flag）跟踪/陷阱标志

当TF=1时,CPU会进入单步模式(single-step mode)CPU在每执行完一条指令后,会自动在该条指令与下条指令之间插入一条int 1h指令并执行它

  • 调试器？
    • 先设置TF=1
    • 跳转到第main
    • 修改int 01h的函数体，控制权到调试器
    • 从而可以看到一些变量值等

没有相关指令，改变FLAG整体才能改变TF位
